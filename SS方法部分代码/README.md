# SS 方法部分代码

这个目录整理了两份基于 Spectral Signatures 思路的后门样本检测实验代码，以及对应的 `.pt` 数据文件。当前内容适合单独阅读、归档和后续整理为公开仓库，但暂不依赖整个大项目的完整流水线。

## 目录结构

```text
SS方法部分代码/
├── SS论文原法代码.py
├── SS增强版代码.py
├── 输入数据说明.md
├── pathmnist_patch.pt
└── cifar10_patch.pt
```

## 文件说明

### `SS论文原法代码.py`

这是一个更接近论文原始思路的基础版谱签名检测实现，主要特点如下：

- 默认输入文件为 `pathmnist_patch.pt`
- 从 `.pt` 文件中读取 `images`、`labels`、`poison_flags`
- 先按标签统计各类别样本数
- 将样本数最多的类别视为 `suspect_class`
- 只在该可疑类别内部执行谱分析
- 使用 `power iteration` 估计主奇异方向
- 使用投影平方作为异常分数
- 使用 Otsu 阈值法执行无监督二值化

核心输出包括：

- 可疑类别 `suspect_class`
- 每个样本的异常分数 `scores`
- 自动阈值 `threshold`
- 预测标签 `y_pred`

### `SS增强版代码.py`

这是一个增强版谱签名实现，相比基础版增加了表示学习与按类别归一化过程，主要特点如下：

- 默认输入文件为 `cifar10_patch.pt`
- 先训练一个小型 CNN 特征提取模型
- 提取倒数第二层的深度特征表示
- 对每个类别分别执行 PCA 低秩分解
- 使用投影能量构造异常分数
- 额外计算类别强度 `class_strength`
- 在每个类别内部使用 `MAD` 进行稳健归一化
- 返回中毒相关类别集合和目标类别掩码

核心输出包括：

- 深度特征 `features`
- 原始异常分数 `raw_scores`
- 归一化分数 `normalized_scores`
- 类别强度 `class_strength`
- 中毒相关类别 `poison_classes`

### `输入数据说明.md`

该文档记录两份脚本使用的数据来源、必需字段和输入预处理规则，是理解 `.pt` 文件结构的重要补充说明。

## 输入数据格式

两个脚本都要求 `.pt` 文件至少包含以下字段：

- `images`
- `labels`
- `poison_flags`

字段含义：

- `images`：图像张量
- `labels`：类别标签
- `poison_flags`：是否为中毒样本的标记

### 基础版输入

- 默认文件：`pathmnist_patch.pt`
- 预处理方式：`images.float() / 255.0`
- 标签类型：`np.int32`

### 增强版输入

- 默认文件：`cifar10_patch.pt`
- 预处理方式：若像素最大值大于 `1`，则除以 `255`
- 标签类型：`torch.long`

## 两种实现的差异

### 论文原法代码

- 更偏向传统 SS 基线
- 不单独训练特征网络
- 在可疑类内部直接对像素展开后进行谱分析
- 结构更简单，便于复现和讲解

### 增强版代码

- 先学习特征表示，再进行谱分析
- 对每个类别分别建模
- 使用 `MAD` 实现跨类别可比较的稳健归一化
- 更适合继续扩展为完整实验脚本

## 运行环境

- Python 3.10+
- PyTorch
- NumPy

增强版通常还需要：

- `torchvision`

## 运行方式

当前两个脚本都没有命令行参数接口，而是通过脚本顶部常量指定输入路径。

### 运行基础版

```bash
python SS论文原法代码.py
```

### 运行增强版

```bash
python SS增强版代码.py
```

## 当前限制

1. 两个脚本都将输入路径写死为旧的 Windows 绝对路径，换机器后需要手动修改。
2. 两个脚本目前主要以 `run()` 或相关函数组织，没有统一的命令行入口和结果保存流程。
3. 增强版代码虽然返回了中间结果，但没有直接输出图表、指标表或结果文件。
4. 基础版默认将“样本数最多的类别”视为可疑类，这是一种经验性简化，不一定适用于所有数据分布。
5. 目录中包含较大的 `.pt` 数据文件，不适合长期直接放在公开 GitHub 仓库中。
